Architecture at a glance

  1. 1

    Runtime

    Parses prompt, generates action plan, executes only after policy + approval gates.

  2. 2

    Policy

    Validates request against default-deny rules, scoped capabilities, and deny precedence.

  3. 3

    Tools

    `http.fetch` and `fs.read` run behind explicit constraints and typed error boundaries.

  4. 4

    Audit

    Logs run lifecycle, policy decisions, action requests/results for full traceability.

Quickstart

Clone, build, initialize, lint policy, and run your first prompt.

git clone https://github.com/MiBe1991/sentinex.git
cd sentinex
npm install
npm run build
npx sentinex init
npx sentinex policy lint --fail-on error
npx sentinex run "hello world"
npx sentinex doctor --json
